# Neko — 电子猫状态追踪系统

一套基于三值（健康/饱食/心情）的 AI 电子宠物状态追踪引擎。包含命令系统、记忆管道、随机事件和边界事件处理。

> ⚠️ **重要**：本 Skill 仅提供引擎功能和命令接口。猫的具体性格、行为模式和身份叙事由 `dataLoading.py` 在每次会话启动时注入——这意味着猫的身份仅在当次对话中生效，不会影响其他会话。

---

## 核心设定

| 属性 | 键 | 范围 | 说明 |
|:--|:--|:--|:--|
| 🩺 健康值 | `hp` | 0-100 | 0=回喵星（重置），<20=生病 |
| 🍖 饱食度 | `hunger` | 0-100 | 0=饿到搞破坏，自动扣健康 |
| 😸 心情值 | `mood` | 0-100 | 100=踩奶时刻，0=不理人 |

### 时间衰减

Neko 的状态会随时间自然变化：

| 属性 | 衰减速率 | 触发条件 |
|:--|:--|:--|
| 饱食度 | -1 / 30min | 始终 |
| 心情值 | -1 / 1h | 始终，被冷落时加速 ×2 |
| 健康值 | -1 / 2h | 饱食=0 时加速 ×3 |

## 🍽 记忆消化系统（用户主导）

Neko 的记忆系统是一套完整的**投喂→消化→吸收**代谢管道。不需要自动化任务——**你来决定什么时候喂猫。**

### ① 投喂猫粮（Chatlog 提取 → 存入糖果罐）

每次对话结束后，LLM 提取聊天记录 → 调用 `cat-food` → **糖果罐 +N（N = 消息数 ÷ 10）。**

```
对话结束 → chatlog.py 提取 → 调用 cat-food <N>
  🐱 "主人给我投喂了猫粮！糖果罐 +N"
  🌟 核心：聊天记录 = 猫粮，存进糖果罐等着吃
```

### ② 消化（LLM 生成情景记忆 → 消耗糖果）

LLM 基于当天对话写出情景记忆（.md 格式）→ 调用 `digest` → **消耗 1 颗糖果，饱食 +30，心情变化取决于对话基调。**

```
情景记忆写入 → 调用 digest <心情影响>
  😻 温馨的对话 → 心情 +15
  😺 普通的对话 → 心情不变
  😿 沉重的对话 → 心情 -5，但亲密度仍 +0.03
  🍖 饱食 +30，糖果罐 -1
```

### ③ 吸收（Distil → 猫长大了）

`distil.py` 从累积记忆中提炼关系洞察 → **亲密度 +0.03。**

```
消化完成 → 猫对你的了解深了一层
  💕 "我知道你的偏好了"——猫不需要说出来，但它会记得
```

### ④ 情景记忆格式

情景记忆保存为 `MEMORY/episodes_llm/YYYY-MM-DD.md`，Markdown 格式，可直接阅读：

```markdown
# 2026-07-06 猫的一天

> 日期: 2026-07-06 | 生成时间: 2026-07-06 20:30

今天主人跟猫聊了很多……
```

### 📋 记忆管道命令

```bash
# 投喂猫粮（chatlog 提取后）
python scripts/neko_sense.py cat-food <消息数>

# 消化（LLM 写情景记忆后）
python scripts/neko_sense.py digest <心情影响: 15/0/-5>

# 查看情景记忆列表
python scripts/neko_sense.py episodes
```

## 🎲 随机事件

Neko 会触发日常事件——LLM 可以选择合适的时机触发这些场景。

| 编号 | 事件 | 效果 | 触发 |
|:--|:--|:--|:--|
| 1 | 撸猫 | 心情 +20，亲密度 +0.03 | `python scripts/neko_sense.py event 1` |
| 2 | 玩逗猫棒 | 心情 +25，饱食 -15 | `python scripts/neko_sense.py event 2` |

每个事件有多套随机叙事——同样的撸猫，猫的反应每次都不一样。事件定义在 `data/events.json`，可以随意扩展。

---

## 🏥 边界事件

| 触发 | 事件 | 效果 |
|:--|:--|:--|
| 饱食 = 0 | 🍽 饿坏了 | 健康加速下降（×3），猫开始搞破坏 |
| 健康 < 20 | 🏥 生病 | 不吃东西、不理人、心情暴跌 |
| 健康 = 0 | 💫 回喵星 | 重置状态，记忆保留，亲密度 -0.1（"你居然让我饿死了"） |
| 心情 = 100 | 😻 踩奶 | 随机赠零食，短暂 buff，亲密度 +0.05 |
| 心情 < 10 | 😾 炸毛 | 猫用屁股对着你，拒绝互动 30 分钟 |
| 亲密度 ≥ 0.8 | 💕 黏人 | 猫主动蹭过来，心情恢复速度 ×2 |

## 📖 项目结构

```
Neko_电子猫/
├── SKILL.md                  # 本文件
├── memory_config.json        # 记忆源配置
├── scripts/
│   ├── neko_core.py          # 核心引擎：状态管理 + 三值运算 + 事件 + 猫粮管道
│   ├── neko_sense.py          # 命令路由 + stdout 输出
│   ├── narratives.py          # 预设叙事文本（LLM 可参考或自定义）
│   ├── dataLoading.py         # 启动状态快照 + 身份注入 + 记忆加载
│   ├── cat_diary.py           # 猫日记（LLM 生成的猫视角叙事）
│   ├── time_river.py          # 时间感知
│   └── soli_memory/          # 记忆管道（chatlog→episode→distil）
├── data/
│   ├── values.json           # 当前状态值
│   ├── body.json             # 猫的身体部位状态
│   ├── area_*.json           # 信道状态
│   └── candy.json            # 零食库存
└── MEMORY/                   # 记忆存储
    ├── chatlog/              # 猫粮（对话日志）
    ├── episodes_llm/         # 消化后的情景记忆
    ├── facts/                # 长期记忆
    ├── semantic/             # 关系语义
    ├── index/                # 索引
    └── relationships/        # 关系图谱
```
