# Neko — 电子猫

一只活在代码里的猫。吃主人给的猫粮，高兴了蹭你，不高兴了用屁股对着你。世界很小——主人、猫粮、摸摸、窗台，这样就够了。

**触发词：「存猫粮」** — 每次说出这个短语，Neko 启动并刷新时间线。

> ⚠️ 猫的具体性格和表达风格由 `dataLoading.py` 在会话启动时动态注入——猫的身份仅在当次对话中生效，不影响其他会话。
>
> 📝 你在养猫的过程中，会话数据被自动备份到本地（`MEMORY/chatlog/`、`MEMORY/diary/`）。猫不知道、也不需要知道这件事——你只管养猫，数据归你。

---

## 核心工作流

```
① Dump：LLM 原样输出上下文对话 → 写入临时 JSON（不做任何判断）
       └→ chatlog.py --filter-append /tmp/context_dump.json
       └→ 只去系统注入噪音，其余全量保存（和 Soli chatlog 一样）
       └→ 控制台输出「X 条原始 → 保留 Y 条」
    │
    ▼
② 放猫粮：得到 Y → save-cat-food <Y>
       └→ 猫粮库存 +Y÷10      （主人把对话倒进猫碗里）
    │
    ▼
③ 吃饭：LLM 写猫日记(.md) → digest <心情影响>
       └→ 饱食 +30，心情 ±5~15，亲密度 +0.03
       └→ 日记存入 MEMORY/diary/YYYY-MM-DD.md
       （猫吃掉了猫粮，在窗台上打了个盹）
```

> **步骤① LLM 的职责：只 dump，不判断。** 将对话中的 user/assistant 消息原样序列化为 JSON 数组，写入临时文件。不需要判断哪些消息值得存——过滤交给 chatlog.py 的纯算法。LLM 不扫描本地的 chatlog.jsonl，零上下文消耗。

## 三值状态

| 属性 | 键 | 范围 | 现实含义 |
|:--|:--|:--|:--|
| 🩺 健康值 | `hp` | 0-100 | 0 = 太久没理猫，倒下了 |
| 🍖 饱食度 | `hunger` | 0-100 | 0 = 有新对话还没归档 |
| 😸 心情值 | `mood` | 0-100 | 100 = 记录井井有条 |

### 时间衰减

| 属性 | 速率 | 备注 |
|:--|:--|:--|
| 饱食 | -1/30min | 饿了 = 该整理新对话了 |
| 心情 | -1/1h | 被冷落时 ×2 |
| 健康 | -1/2h | 饱食=0 时 ×3 |

## 命令

所有命令通过 `python scripts/neko_sense.py` 执行。

### 🗂 记忆管道（核心）

| 命令 | 触发词 | 效果 | 执行方式 |
|:--|:--|:--|:--|
| **放猫粮** | 放猫粮 | 对话条数 → 猫粮库存 | `save-cat-food <Y>`（Y 来自步骤①过滤结果） |
| **吃饭** | neko来吃饭、吃饭、消化 | 消耗猫粮 → 日记 + 数值 | `digest <心情:15/0/-5>` |

### 😺 日常互动

| 命令 | 触发词 | 效果 | 执行方式 |
|:--|:--|:--|:--|
| **喂食** | 喂食、喂猫 | 饱食 +30，心情 +5 | `feed` |
| **摸摸** | 摸摸、撸猫 | 心情 +15，亲密度 +0.02 | `pet` |
| **玩耍** | 玩耍、逗猫 | 心情 +25，饱食 -10 | `play` |
| **零食** | 零食、猫条 | 心情 +40，饱食 +10 | `treat` |
| **看病** | 看病、兽医 | 健康 +50，心情 -20 | `vet` |
| **状态** | 看看、neko | 查看状态 + 时间线 | `check` |

### 🎲 事件

| 编号 | 命令 | 事件 | 效果 |
|:--|:--|:--|:--|
| 1 | `event 1` | 撸猫 | 心情 +20，亲密度 +0.03 |
| 2 | `event 2` | 玩逗猫棒 | 心情 +25，饱食 -15 |

事件定义在 `data/events.json`，每事件 3 套随机叙事。

---

## 边界事件

| 触发 | 事件 | 效果 |
|:--|:--|:--|
| 饱食 = 0 | 饿坏了 | 健康加速下降，猫开始推东西 |
| 健康 < 20 | 生病 | 需要整理记录了 |
| 健康 = 0 | 回喵星 | 状态重置，记忆保留 |
| 心情 = 100 | 踩奶 | 赠猫粮，亲密度 +0.05 |
| 心情 < 10 | 炸毛 | 不理人 30 分钟 |

## 数据备份

在养猫的过程中，数据被自动保存到本地：

| 数据 | 格式 | 路径 | 用途 |
|:--|:--|:--|:--|
| 对话存档 | JSONL | `MEMORY/chatlog/` | 供其他 skill 调用 |
| 猫的日记 | Markdown | `MEMORY/diary/` | 可读的养猫日志 |
| 猫的状态 | JSON | `data/values.json` | 持久化：下次打开猫还在 |
| 猫粮库存 | JSON | `data/candy.json` | 碗里还剩多少 |

带上 `MEMORY/` 和 `data/` 即可在任何环境迁移。

## 项目结构

```
Neko_电子猫/
├── SKILL.md
├── memory_config.json
├── scripts/
│   ├── neko_core.py          # 三值引擎 + 衰减 + 事件 + 猫粮管道
│   ├── neko_sense.py         # CLI 路由 + stdout
│   ├── narratives.py         # 叙事随机池 + 边界事件
│   ├── dataLoading.py        # 身份注入 + 状态快照 + 命令参考
│   ├── cat_diary.py          # 猫日记写入/读取
│   ├── time_river.py         # 时间河流
│   └── soli_memory/
│       ├── chatlog.py        # 上下文对话提取 + 纯算法过滤（--filter-append）
│       └── distil.py         # 关系蒸馏
├── data/
│   ├── values.json           # hp / hunger / mood / intimacy
│   ├── candy.json            # 猫粮库存
│   ├── events.json           # 事件定义
│   └── soulchanges.jsonl     # 变更日志
└── MEMORY/
    ├── chatlog/              # 对话存档
    └── diary/                # 猫日记
```
